## Description
Two backend files have problematic exception handling:
1. `restx_api/margin.py` (line 63): Uses bare `except:` which catches ALL exceptions including `SystemExit` and `KeyboardInterrupt`
2. `services/gex_service.py` (lines 102-103, 125-126): Uses `except Exception: pass` which silently swallows errors

### Fix 1: margin.py

```python

# Before
except: error_response = {"status": "error", "message": "Failed to fetch margin data"}

# After
except Exception as e: logger.exception(f"Failed to fetch margin data: {e}") error_response = {"status": "error", "message": "Failed to fetch margin data"} ```

### Fix 2: gex_service.py
```python

# Before
except Exception: pass

# After
except Exception as e: logger.warning(f"Failed to process option data: {e}") ```

## Acceptance Criteria
-  No bare `except:` clauses remain
-  All `except Exception: pass` replaced with logging
-  Errors are captured with `as e` and logged

## Skills You'll Learn
- Python exception handling best practices
- Why bare except is dangerous (PEP 8, B001)
- Proper error logging patterns
